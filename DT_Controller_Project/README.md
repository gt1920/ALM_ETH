# DT_Controller_Project — PC Control Software

WinForms GUI in C# / .NET Framework **4.8** for managing ALM laser-driver devices. Talks to:
- **ETH variant**: TCP `port 40000` to [ALM_CIC_APP](../ALM_CIC_APP/) on STM32H563
- **USB variant**: HID `(VID 0x0483, PID 0xA7D1)` to [ALM_USB_Project](../ALM_USB_Project/) on STM32G0B1

Discovers ETH devices via UDP broadcast (`port 40001`, magic `"DTDR"`).

## Features

- **Device discovery**: Listens for UDP `"DTDR"` broadcasts; populates ETH device list. USB devices arrive via `WM_DEVICECHANGE → DBT_DEVICEARRIVAL`.
- **Single TCP client constraint**: ETH devices only accept one connection — UI disconnects existing control connection before starting an OTA.
- **Right-click context menus**:
  - ETH device → **Upgrade** → opens `.cic` file dialog → START / DATA / END
  - Module entry → **Upgrade** → opens `.mot` file dialog → CMD_MODULE_UPGRADE relay
- **OTA UI behavior**:
  - `null` END response = success (device already rebooting)
  - All firmware-validation rejections show the same generic message: **"Upgrade failed: firmware invalid."**
  - Full UI refresh after both `.cic` and `.mot` upgrades complete (commit `c4ae801`)
  - Auto-stops `GET_MODULE_INFO` polling spam after Module clicked (commit `f2b9f62`)
- **Locale safety**: All runtime string literals are ASCII — no Chinese byte sequences (cross-locale safe).

## TCP Frame Format (must match firmware)

```
[LEN_L][LEN_H][0xA5][CMD][DIR][SEQ_L][SEQ_H][LEN_BODY][SUBCMD][payload...]
```

| CMD | Value | Use |
|---|---|---|
| `CMD_INFO` | `0x01` | Version query, module list, device name |
| `CMD_MOTION` | `0x10` | Motion command |
| `CMD_PARAM_SET` | `0x20` | Set motor parameter |
| `CMD_UPGRADE` | `0x30` | CIC self-OTA |
| `CMD_MODULE_UPGRADE` | `0x31` | Motor module OTA relay |
